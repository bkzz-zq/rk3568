### 1.2 为什么采用这种架构

| 传统方案 | 本项目方案 |
|---------|-----------|
| RK3568 编码 JPEG + Base64 推送 | RK3568 只发 JSON（~1KB/帧） |
| PC 显示压缩后的低画质画面 | PC 直连 RTSP 原始画质 |
| RK3568 CPU 占用高（编码） | RK3568 CPU 占用低（纯检测） |
| 帧率 ~8 FPS | 帧率 ~10 FPS（持续优化） |

## 2. RK3568 NPU 详细配置

### 2.1 硬件信息

| 项目 | 参数 |
|------|------|
| SoC | RK3568 |
| CPU | 4 核 ARM Cortex-A55 @ 2.0GHz |
| NPU | 0.8 TOPS (INT8) |
| RAM | 建议 2GB+ |
| 架构 | aarch64 (ARM64) |

### 2.2 NPU 软件栈版本（关键！版本必须匹配）

| 组件 | 版本 | 说明 |
|------|------|------|
| **rknn-toolkit2** | 2.3.2 | PC 端模型转换工具（x86_64） |
| **rknn-toolkit-lite2** | 2.3.2 | 板子端推理库（aarch64） |
| **librknnrt.so** | 2.3.2 | NPU 驱动运行时 |
| **RKNN 驱动** | 0.9.0 | 内核驱动（固件自带） |

> ⚠️ **版本必须完全一致！** 如果 rknn-toolkit2 用 2.3.2 转换模型，板子上的 librknnrt.so 也必须是 2.3.2，否则会报错。

### 2.3 版本检查方法

```bash
# 板子上检查运行时版本
python3 -c "
from rknnlite.api import RKNNLite
rknn = RKNNLite()
print('RKNNLite version:', rknn._version)
"

# 检查驱动版本（从日志看）
# I RKNN: RKNN Driver Information, version: 0.9.0
# I RKNN: RKNN Runtime Information, librknnrt version: 2.3.2

# 检查 .so 文件
strings /usr/lib/librknnrt.so | grep "version"
```

### 2.4 NPU 工作原理

```
PC 端（x86_64）                          板子端（aarch64）
┌──────────────┐                        ┌──────────────┐
│ rknn-toolkit2│                        │rknn-toolkit  │
│              │                        │   -lite2     │
│ ONNX/PyTorch │    .rknn 文件           │              │
│    ↓         │ ──────────────────────► │ RKNNLite()  │
│ RKNN 模型     │       传输              │    ↓         │
│ (.rknn)      │                        │ NPU 推理      │
│              │                        │ (硬件加速)    │
└──────────────┘                        └──────────────┘
```

**NPU 只做推理（inference），不做训练。** 流程：
1. PC 上用 `rknn-toolkit2` 将 ONNX → RKNN 格式
2. 将 `.rknn` 文件传到板子
3. 板子用 `rknn-toolkit-lite2` 加载模型，调用 NPU 推理

### 2.5 librknnrt.so 安装位置

```bash
# 默认位置
/usr/lib/librknnrt.so
/usr/lib/librknnrt.so.2

# 如果手动更新
cp librknnrt.so /usr/lib/
cp librknnrt.so /usr/lib/aarch64-linux-gnu/
ldconfig
```

## 3. YOLOv8 → RKNN 模型转换

### 3.1 完整转换流程（PC 端执行）

```python
"""
convert_model.py - 在 PC (x86_64) 上执行
将 YOLOv8 ONNX 模型转换为 RKNN 格式
"""

from rknn.api import RKNN

# 1. 创建 RKNN 对象
rknn = RKNN()

# 2. 配置模型
#    mean_values: 输入归一化均值
#    std_values: 输入归一化标准差
#    注意：YOLOv8 输入是 0-255 uint8，NPU 内部做归一化
rknn.config(
    mean_values=[[0, 0, 0]],       # YOLOv8 不需要减均值
    std_values=[[255, 255, 255]],   # 除以 255 → 0~1
    target_platform='rk3568'        # 指定目标平台
)

# 3. 加载 ONNX 模型
ret = rknn.load_onnx(model='yolov8n.onnx')
if ret != 0:
    print('Load ONNX failed!')
    exit(-1)

# 4. 构建 RKNN 模型
#    do_quantization=True: 启用量化（INT8，提高 NPU 速度）
#    dataset.txt: 量化校准数据集（几张代表性图片的路径）
ret = rknn.build(
    do_quantization=True,
    dataset='./dataset.txt'  # 需要准备！
)
if ret != 0:
    print('Build RKNN failed!')
    exit(-1)

# 5. 导出
rknn.export_rknn('./yolov8n.rknn')

# 6. 精度测试（可选）
ret = rknn.init_runtime(target=None)  # PC 上模拟测试
# ...

rknn.release()
```

### 3.2 量化校准数据集准备

```bash
# dataset.txt - 每行一张图片的绝对路径
# 用 10-50 张代表性图片即可
/path/to/calibration/image1.jpg
/path/to/calibration/image2.jpg
/path/to/calibration/image3.jpg
...
```

可以用 ffmpeg 从摄像头抓取：
```bash
ffmpeg -i "rtsp://admin:password@192.168.0.66:554/Streaming/Channels/201" \
       -vf "fps=1" -frames:v 20 calibration/%03d.jpg
```

### 3.3 模型输入/输出规格

| 项目 | 规格 |
|------|------|
| 输入尺寸 | 1x3x640x640 (NCHW, static shape) |
| 输入格式 | uint8 (0-255)，NPU 内部归一化 |
| 输出格式 | float32 [1, 84, 8400] |
| 84 维含义 | 4(cxcywh) + 80(classes) |
| 8400 个检测 | 不同尺度的 anchor 预测 |

### 3.4 不量化也能跑



### 4.2 踩坑记录

#### 坑1：版本不匹配

**现象：**
```
E RKNN: load rknn model fail!
```

**原因：** PC 用 2.x 转换的模型，板子 librknnrt.so 是 1.x

**解决：** 确保所有组件版本一致（2.3.2）

#### 坑2：librknnrt.so 权限

**现象：**
```
ImportError: librknnrt.so: cannot open shared object file
```

**解决：**
```bash
cp librknnrt.so /usr/lib/
ldconfig
```

#### 坑3：GPU device discovery warning

**现象：**
```
[W:onnxruntime:Default, device_discovery.cc:164 DiscoverDevicesForPlatform]
GPU device discovery failed
```

**原因：** HyperLPR3 使用的 onnxruntime 尝试找 GPU

**影响：** 无影响，可忽略

#### 坑4：query dynamic range warning

**现象：**
```
W RKNN: query RKNN_QUERY_INPUT_DYNAMIC_RANGE error
W Query dynamic range failed. Ret code: RKNN_ERR_MODEL_INVALID
```

**原因：** 静态 shape 模型不支持动态输入

**影响：** 无影响，日志已说明 "please ignore the above warning"

## 5. NPU 行人检测代码详解

### 5.1 核心代码（src/detector/person.py）

```python
from rknnlite.api import RKNNLite

class PersonDetector:
    def __init__(self, config, npu_enabled=False):
        self.model = None
        self.input_size = (640, 640)
        self.npu_enabled = npu_enabled

    def _load_rknn_model(self):
        """加载 RKNN 模型"""
        self.model = RKNNLite()

        # ① 加载 .rknn 模型文件
        ret = self.model.load_rknn("models/yolov8n.rknn")
        if ret != 0:
            return False

        # ② 初始化运行时（target=None 自动选择 NPU）
        ret = self.model.init_runtime(target=None)
        if ret != 0:
            return False

        return True

    def _detect_rknn(self, frame):
        """NPU 推理"""
        # ① letterbox 缩放（保持比例）
        img, ratio, (dw, dh) = letterbox(frame, self.input_size)

        # ② BGR → RGB
        img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)

        # ③ 添加 batch 维度，确保 uint8 类型
        #    NPU 要求输入: (1, H, W, C) uint8
        img = np.expand_dims(img, 0).astype(np.uint8)

        # ④ NPU 推理
        outputs = self.model.inference(inputs=[img])

        # ⑤ 后处理（NMS、坐标还原）
        return self._postprocess_yolo(outputs[0], frame.shape, ratio, dw, dh)
```

### 5.2 关键注意事项

#### ⚠️ 输入格式必须正确

```python
# 错误！3 维输入会报错
img = np.array(img, dtype=np.uint8)  # shape: (640, 640, 3)

# 正确！4 维输入
img = np.expand_dims(img, 0).astype(np.uint8)  # shape: (1, 640, 640, 3)
```

**RKNNLite 的 `inference()` 要求 4 维输入：`(batch, height, width, channels)`**

#### ⚠️ 不要手动归一化

```python
# 错误！NPU 内部会做归一化（基于 config 中的 mean_values/std_values）
img = img.astype(np.float32) / 255.0

# 正确！直接传 uint8
img = np.expand_dims(img, 0).astype(np.uint8)
```

#### ⚠️ letterbox 的用途

```
原始帧 1920x1080 → letterbox → 640x640（上下填充黑边）
                                    ↑
                              NPU 输入尺寸

检测坐标需要还原：
原图坐标 = (检测坐标 - padding) / ratio
```

## 6. 车牌识别（HyperLPR3 + ROI）

### 6.1 HyperLPR3 工作方式

```python
from hyperlpr3 import LicensePlateCatcher

catcher = LicensePlateCatcher()
results = catcher(frame)

# 返回格式: [(号牌, 置信度, 类型, [x1,y1,x2,y2]), ...]
# 类型: 0=蓝牌, 1=黄牌, 2=白牌, 3=绿牌
# 例如: [('湘AU5555', 0.991, 0, [425, 239, 685, 335])]
```

### 6.2 ROI 区域裁剪优化

```python
def _crop_roi(self, frame):
    """
    ROI 配置: config.yaml → plate_recognition.roi: [0.1, 0.4, 0.9, 0.95]
    归一化坐标: [左边10%, 上边40%, 右边90%, 下边95%]

    好处:
    1. 减少搜索区域 → HyperLPR3 更快
    2. 避免误检
    3. 车牌通常在画面下半部分
    """
    h, w = frame.shape[:2]
    x1 = int(roi[0] * w)  # 192
    y1 = int(roi[1] * h)  # 432
    x2 = int(roi[2] * w)  # 1728
    y2 = int(roi[3] * h)  # 1026

    roi_frame = frame[y1:y2, x1:x2]  # 1536x594
    return roi_frame, (x1, y1)
```

### 6.3 坐标映射

```python
# ROI 区域内的检测坐标需要映射回原图
if roi_offset:
    for r in results:
        r["bbox"] = [
            r["bbox"][0] + roi_offset[0],  # x + offset_x
            r["bbox"][1] + roi_offset[1],  # y + offset_y
            r["bbox"][2] + roi_offset[0],
            r["bbox"][3] + roi_offset[1],
        ]
```

## 8. 性能优化记录

### 8.1 优化历程

| 阶段 | 优化措施 | FPS |
|------|---------|-----|
| 初始 | 全帧检测 + JPEG编码推送 | ~3 |
| 第1轮 | 缩放到640px + 每3帧检测 | ~8 |
| 第2轮 | 车牌降频每9帧 + 去掉图像编码 | ~10 |
| 第3轮 | NPU行人 + RTSP直连分离架构 | ~10 |
| 当前 | ROI裁剪 + 并行检测 | 8-11 |

### 8.2 性能瓶颈分析（RK3568 4核分配）

| 核心 | 任务 | 耗时 |
|------|------|------|
| 核0 | 主循环（读帧+推送JSON） | ~10ms |
| 核1 | NPU 行人检测（每3帧） | ~10ms |
| 核2 | HyperLPR3 车牌检测（每9帧） | ~150-400ms |
| 核3 | 备用 | - |

### 8.3 当前帧率

- **无车牌场景**: ~10 FPS（NPU行人 ~10ms + 读帧 ~30ms）
- **有车牌场景**: ~8 FPS（NPU行人 + HyperLPR3 ~400ms 并行）
- **无客户端连接**: 检测仍运行（为 RTSP 直连架构服务）

## 10. 常见问题与排查

### 10.1 NPU 相关

**Q: RKNN 模型加载失败**
```
检查：
1. .rknn 文件路径是否正确
2. librknnrt.so 版本是否与模型匹配（都是 2.3.2）
3. NPU 驱动是否正常: dmesg | grep rknpu
```

**Q: NPU 推理报维度错误**
```
原因：输入不是 4 维
解决：img = np.expand_dims(img, 0).astype(np.uint8)
确保 shape 为 (1, 640, 640, 3)
```

**Q: NPU 检测不到行人**
```
排查：
1. 确认输入是 RGB（不是 BGR）: cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
2. 确认没有手动归一化（NPU 内部做）
3. 确认 letterbox 后尺寸是 640x640
4. 降低置信度阈值: config.yaml → person_detection.confidence: 0.3
```

### 10.2 车牌识别相关

**Q: HyperLPR3 始终返回 0**
```
排查：
1. 确认画面中确实有车牌（保存测试帧检查）
2. 尝试不同分辨率（640x360 / 960x540 / 1920x1080）
3. 检查 ROI 区域是否正确
4. 查看日志中的 "原始检测: X 个结果"
```

**Q: 车牌号显示问号**
```
原因：OpenCV putText 不支持中文
解决：PC 客户端用 PIL 渲染中文文字
```

**Q: 有车牌时帧率很低**
```
原因：HyperLPR3 检测耗时 ~400ms
优化：
1. 增大 PLATE_DETECT_INTERVAL（如 15 或 20）
2. 缩小 ROI 区域
3. 长期方案：替换为 YOLOv8 车牌检测 + LPRNet（NPU 加速）
```

### 10.3 视频流相关

**Q: RTSP 连接失败**
```
检查：
1. 摄像头 IP 是否可达: ping 192.168.0.66
2. RTSP 地址是否正确（主码流101/子码流201）
3. 用户名密码是否正确
4. 网络是否通畅
```

**Q: 画面卡顿/延迟大**
```
优化：
1. 设置缓冲区大小: cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)
2. 使用子码流（分辨率更低）
3. PC 客户端直连 RTSP（不走板子转发）
```

## 11. 移植指南

### 11.1 移植到其他 RK3568 板子

1. **安装系统**：确保内核包含 NPU 驱动（`dmesg | grep rknpu`）
2. **安装 Python 3.10+**
3. **安装 RKNNLite2**：`pip install rknn_toolkit_lite2-2.3.2-...whl`
4. **更新 librknnrt.so**：`cp librknnrt.so /usr/lib/ && ldconfig`
5. **上传模型文件**：`models/yolov8n.rknn`
6. **克隆代码**：`git clone https://github.com/bkzz-zq/rk3568.git`
7. **安装依赖**：`pip install -r requirements.txt`
8. **修改配置**：`config/config.yaml`（RTSP 地址等）
9. **启动测试**：`python3 src/main.py --no-ocr`

### 11.2 移植到 RK3588/RK3588S

RK3588 有 6 TOPS NPU（比 RK3568 的 0.8 TOPS 强很多），步骤类似：

1. 修改 `target_platform='rk3588'`（模型转换时）
2. 安装对应的 `rknn_toolkit_lite2`（同一版本 2.3.2 通用）
3. 性能提升约 5-7 倍

### 11.3 添加新的 NPU 检测模型

以添加车辆检测为例：

1. **PC 端**：准备 ONNX 模型，用 `convert_model.py` 转换为 RKNN
2. **创建检测器**：复制 `person.py` 为 `vehicle.py`
3. **修改类别**：COCO 中 car=2, truck=7, bus=5
4. **在 main.py 注册**：添加 VehicleDetector 并加入并行检测
5. **更新推送**：在 push_results 中添加 vehicles 字段

### 11.4 替换车牌识别为 NPU 方案（长期规划）

```
当前：HyperLPR3（CPU ~400ms）
目标：YOLOv8 车牌检测（NPU ~10ms）+ LPRNet 识别（CPU ~10ms）

步骤：
1. 训练 YOLOv8 车牌检测模型（CCPD 数据集）
2. 转换为 RKNN 格式
3. 集成 LPRNet 轻量级车牌识别模型
4. 替换 plate.py 中的 HyperLPR3
```

### RKNNLite API

```python
from rknnlite.api import RKNNLite

rknn = RKNNLite()
rknn.load_rknn(model_path)     # 加载模型，返回 0=成功

outputs = rknn.inference(inputs=[img])  # 推理，返回输出列表
rknn.release()                  # 释放资源
```
