## 1. 快速移植清单

移植到新环境只需关注以下几项：

```
□ 1. Python 3.8+ 环境
□ 2. 摄像头 RTSP 地址（config.yaml → camera）
□ 3. NPU 驱动和运行时（如果有 NPU）
□ 4. OpenCV + GStreamer 硬解码（如果用 RTSP）
□ 5. 中文字体（视频流显示车牌号）
□ 6. MQTT Broker（如果用智能家居功能）
□ 7. 内网穿透（如果需要外网访问）
```

## 2. 更换硬件平台

### 2.1 同型号 RK3568（不同厂商板卡）

大部分 RK3568 板卡通用，差异主要在：

| 差异点 | 说明 | 处理方式 |
|--------|------|---------|
| NPU 驱动 | 内核驱动版本可能不同 | 安装对应版本的 `librknnrt.so` |
| 系统包 | 预装软件不同 | 运行 `deploy/deploy.sh` 自动安装 |
| GStreamer | 有些板卡没预装 | `apt install gstreamer1.0-rockchip` |
| Python 版本 | 可能是 3.8 / 3.9 / 3.10 | 虚拟环境自动处理 |

**步骤**：

```bash
# 1. 传输项目
scp -r rk3568/ root@<新板子IP>:/opt/rk3568

# 2. 运行部署脚本
cd /opt/rk3568/deploy
chmod +x deploy.sh && ./deploy.sh

# 3. 修改配置（主要是摄像头地址）
vi /opt/rk3568/config/config.yaml

# 4. 启动测试
systemctl start rk3568
```

### 2.2 RK3588 / RK3588S

RK3588 性能更强（6 TOPS NPU），代码无需修改，只需：

```bash
# 1. 安装 RK3588 的 NPU 运行时（与 RK3568 不同）
wget https://github.com/rockchip-linux/rknpu2/raw/master/runtime/Linux/librknn_api/aarch64/librknnrt.so
sudo cp librknnrt.so /usr/lib/
sudo ldconfig

# 2. 修改 config.yaml 中的 NPU 目标
npu:
  target: "rk3588"   # 改为 rk3588

# 3. 重新转换 RKNN 模型（RK3588 的模型不通用）
# 在 PC 上：
python convert_model.py --target rk3588
# 将生成的 .rknn 文件传到板子的 models/ 目录
```

### 2.3 RK3566

与 RK3568 几乎相同，NPU 算力稍低（1 TOPS）：

```yaml
# config.yaml
npu:
  target: "rk3566"
```

### 2.4 非 Rockchip 平台（树莓派、Jetson 等）

不支持 NPU，使用纯 CPU 模式：

```yaml
# config.yaml
npu:
  enabled: false
```

参见 [第 5 节：纯 CPU 模式](#5-纯-cpu-模式无-npu)

## 3. 更换摄像头

### 3.1 海康威视（Hikvision）

```
主码流: rtsp://用户名:密码@IP:554/Streaming/Channels/101
子码流: rtsp://用户名:密码@IP:554/Streaming/Channels/201
```

### 3.2 大华（Dahua）

```
主码流: rtsp://用户名:密码@IP:554/cam/realmonitor?channel=1&subtype=0
子码流: rtsp://用户名:密码@IP:554/cam/realmonitor?channel=1&subtype=1
```

### 3.3 通用 ONVIF 摄像头

用 ONVIF 协议发现设备获取 RTSP 地址：
```bash
# 安装 onvif 工具
pip install onvif-zeep

# 或在摄像头 Web 管理页面查看 RTSP 地址
```

### 3.4 USB 摄像头

修改 `config.yaml`：
```yaml
camera:
  # 改为 USB 设备号
  rtsp: ""
  usb_device: 0    # /dev/video0
```

> 注意：USB 摄像头不支持 GStreamer 硬解码，会自动回退到 OpenCV 软解码。

### 3.5 配置建议

| 场景 | use_stream | 分辨率 | 说明 |
|------|-----------|--------|------|
| 局域网 + NPU | sub | 640×360 | 推荐，性能最佳 |
| 局域网 + CPU | sub | 640×360 | CPU 占用低 |
| 远程监控 | sub | 640×360 | 带宽占用低 |
| 需要高清画面 | main | 1920×1080 | NPU 可处理，CPU 模式会很卡 |

## 4. 配置文件详解

> `config/config.yaml` 所有可配置项说明

### 摄像头

```yaml
camera:
  main_stream: "rtsp://..."      # 主码流（高清）
  sub_stream: "rtsp://..."       # 子码流（标清）
  use_stream: "sub"              # 使用的码流: "main" 或 "sub"
  fps_limit: 15                  # 帧率限制（0=不限）
  reconnect_interval: 5          # 断线重连间隔（秒）
  hw_decode: true                # GStreamer 硬解码（不可用自动回退）
```

**移植时必改**：`main_stream`、`sub_stream` 改为你的摄像头地址

### 行人检测

```yaml
person_detection:
  enabled: true                  # 是否启用
  confidence: 0.5                # 检测置信度阈值（越低检出越多，误检也多）
  nms_threshold: 0.4             # NMS 阈值（一般不用改）
  model_path: "models/yolov8n.rknn"   # RKNN 模型（NPU 用）
  onnx_model_path: "models/yolov8n.onnx"  # ONNX 模型（CPU 调试用）
```

**移植时注意**：
- 无 NPU：确保 `npu.enabled: false`，程序会用 ONNX 模型
- 换平台：需要重新转换 RKNN 模型

### 车牌识别

```yaml
plate_recognition:
  enabled: true
  confidence: 0.5
  roi: [0.1, 0.4, 0.9, 0.95]    # 检测区域（归一化坐标）
```

**ROI 调整**：`[左, 上, 右, 下]`，值域 0.0~1.0
- 摄像头正对大门：`[0.1, 0.4, 0.9, 0.95]`（下半部分）
- 摄像头俯拍停车场：`[0.0, 0.0, 1.0, 1.0]`（全画面）
- 车道在左侧：`[0.0, 0.3, 0.5, 0.95]`

### NPU

```yaml
npu:
  enabled: true                  # false=纯 CPU 模式
  target: "rk3568"               # NPU 目标平台
```

### 智能家居（可选）

```yaml
iot:
  enabled: true                  # false=关闭智能家居功能
  mqtt:
    host: "127.0.0.1"            # MQTT Broker 地址
    port: 1883
  server:
    host: "0.0.0.0"              # HTTP 服务器监听地址
    port: 8000                   # HTTP 端口
  sensor_collect_interval: 180   # 传感器采集间隔（秒）
```

**移植时注意**：
- 不需要智能家居：设 `enabled: false`
- MQTT Broker 不在本机：改 `host`
- 端口冲突：改 `port`

## 5. 纯 CPU 模式（无 NPU）

适用于：树莓派、Jetson、普通 PC、云服务器

### 配置

```yaml
npu:
  enabled: false
```

### 性能对比

| 平台 | 行人检测 | 车牌识别 | 总 FPS |
|------|---------|---------|--------|
| RK3568 + NPU | ~30ms (NPU) | ~150ms (CPU) | ~25fps |
| RK3568 纯 CPU | ~200ms | ~150ms | ~8fps |
| 树莓派 4B | ~500ms | ~300ms | ~3fps |
| PC (i5) | ~50ms | ~80ms | ~20fps |

### CPU 模式优化建议

1. 增大检测间隔（`main.py` 中修改）：
   ```python
   PERSON_INTERVAL = 9    # 每 9 帧检测一次（原 3）
   PLATE_INTERVAL = 9     # 每 9 帧检测一次（原 3）
   ```

2. 降低检测分辨率：
   ```python
   DETECT_WIDTH = 320     # 从 640 降到 320
   ```

3. 关闭不需要的模块：
   ```bash
   python3 src/main.py --no-plate    # 关闭车牌识别
   python3 src/main.py --no-person   # 关闭行人检测
   ```

## 6. 不同平台依赖安装

### RK3568 / RK3588（aarch64）

```bash
# 一键部署
cd /opt/rk3568/deploy && ./deploy.sh

# 或手动安装
pip install "numpy<2" opencv-python pyyaml websockets
pip install rknnlite2          # NPU 运行时
pip install hyperlpr3          # 车牌识别
pip install fastapi uvicorn paho-mqtt PyJWT Pillow
```

### 树莓派（aarch64 / armv7l）

```bash
sudo apt install python3-venv libopencv-dev
python3 -m venv .venv && source .venv/bin/activate
pip install numpy opencv-python pyyaml websockets
pip install hyperlpr3
pip install fastapi uvicorn paho-mqtt PyJWT Pillow
# 无 NPU，不装 rknnlite2
```

### x86 PC / 服务器

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install numpy opencv-python pyyaml websockets
pip install ultralytics        # YOLOv8 CPU 推理
pip install hyperlpr3
pip install fastapi uvicorn paho-mqtt PyJWT Pillow
```

### Docker（任意平台）

```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python3", "src/main.py", "--no-npu"]
```

> 注意：Docker 内无法使用 NPU 和 GStreamer 硬解码

## 7. 网络与外网配置

### 局域网直连

```
App/PC → http://RK3568_IP:8000
```

无需额外配置。

### 外网访问（cpolar 内网穿透）

```bash
# 安装 cpolar
curl -L https://www.cpolar.com/static/downloads/install-release-cpolar.sh | sudo bash

# 启动穿透（后台运行）
cpolar http 8000 -sd=your_subdomain

# 获取外网地址
cpolar status
# 显示: https://xxxx.cpolar.top → http://localhost:8000
```

**App 配置**：将服务器地址设为 cpolar 提供的 HTTPS 地址

### MQTT 外网

如果 STM32 设备不在局域网内：
1. 使用公共 MQTT Broker（如 EMQX 公共服务器）
2. 或通过 cpolar 穿透 1883 端口

## 8. 验证移植成功

### 步骤 1：检查服务启动

```bash
systemctl status rk3568
# 应显示 active (running)

journalctl -u rk3568 -n 20 --no-pager
# 应看到：
# ✓ 行人检测模块就绪
# ✓ 车牌识别模块就绪
# ✓ 智能家居 IoT 服务已启动
```

### 步骤 2：检查摄像头

```bash
# 日志中应显示摄像头连接成功
# ✓ 硬解码连接成功: 640x360 @ 25.0fps [GStreamer+MPP]
```

如果失败：
```bash
# 测试 RTSP 连接
ffprobe "rtsp://admin:password@IP:554/Streaming/Channels/201"
```

### 步骤 3：检查 NPU

```bash
# 日志中应显示
# RKNN 模型加载成功: models/yolov8n.rknn

# 手动测试
python3 -c "from rknnlite.api import RKNNLite; print('NPU OK')"
```

### 步骤 4：检查 API

```bash
# 健康检查
curl http://localhost:8000/health
# {"status":"ok","mqtt_connected":true,...}

# 登录测试
curl -X POST http://localhost:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
# {"success":true,"token":"eyJ..."}
```

### 步骤 5：检查视频流

浏览器打开 `http://RK3568_IP:8000/video/stream`，应看到实时画面。

### 步骤 6：检查车牌识别

```bash
# 日志中应交替出现
# 车牌识别耗时: xxxms, 识别到 0 个车牌
# 车牌识别耗时: xxxms, 识别到 1 个车牌  ← 有车牌时
```

### 常见移植问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `import cv2` 失败 | 缺少 OpenCV | `pip install opencv-python` |
| GStreamer 不可用 | pip 版 OpenCV 无 GStreamer | 链接系统 OpenCV（见部署文档） |
| NPU 初始化失败 | 驱动/运行时版本不匹配 | 重装对应版本 `librknnrt.so` |
| 车牌始终 0（systemctl） | 环境变量缺失 | 检查 service 文件中 Environment |
| MQTT 连接失败 | mosquitto 未安装 | `apt install mosquitto` |
| 中文显示问号 | 缺少中文字体 | `apt install fonts-wqy-microhei` |
| 端口被占用 | 8000/8765 已被使用 | 修改 config.yaml 中的端口 |
| 内存不足 OOM | 板卡内存太小 | 用子码流 + 降低检测频率 |
